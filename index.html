📌 What is a Function?

🧱 Organized

A function is a block of organized, reusable code that performs a single, related action.

🔁 Reusable

Functions provide a high degree of code reusing — write once, call many times.

🧩 Modular

Functions provide better modularity for applications by breaking code into logical pieces.

def function_name(parameters):

greet(name="Alice")

def my_func(*args):  # receives tuple
    pass

# **kwargs — arbitrary keyword
def my_func(**kwargs):  # receives dict
    pass

📖 All Topics
1️⃣ Defining a Function
📌
Use the def keyword to define a function. The function body must be indented. Defining a function does NOT execute it — it only registers the block of code.
# Simplest function definition
def my_function():
    print("Hello From My Function!")

# Function with a parameter
def func1():
    print("I am learning Python Function")

# Calling it:
func1()   # Output: I am learning Python Function
💡 Key Rules for Defining Functions
1
Must start with the def keyword
2
Function name must follow Python identifier rules (no spaces, no starting with numbers)
3
Parentheses () are required even with no parameters
4
A colon : ends the function header
5
The function body must be indented (4 spaces or 1 tab)
2️⃣ Calling a Function
📌
Once a function is defined, you can call it from another function, a program, or the Python prompt. Simply write the function name followed by parentheses.
def my_function():
    print("Hello From My Function!")

# Calling the function — executes its body
my_function()
# Output: Hello From My Function!

# Can be called multiple times
my_function()
my_function()
my_function()
⚠️ Common Mistake
❌ Wrong
# This just references the function object
# (does NOT call it)
my_function   # no parentheses!
✅ Correct
# This CALLS the function
my_function()  # parentheses required!
3️⃣ Arguments and Parameters

📥 Parameter Definition side

The variable listed inside the parentheses in the function definition. It's a placeholder.

📤 Argument Call side

The actual value passed when calling the function. It fills in the parameter.

def my_function(fname):         # fname is the PARAMETER
    print(f"Hello, {fname}!")

my_function("Alice")               # "Alice" is the ARGUMENT
my_function("Bob")                 # "Bob" is the ARGUMENT
# Output:
# Hello, Alice!
# Hello, Bob!

# Another example with greet:
def greet(name):
    print(f"Hello, {name}!")

greet("Alice")   # Output: Hello, Alice!
4️⃣ Return Values
📌
Use the return statement to send a value back from a function. The return statement also exits the function block immediately.
def my_function(x):
    return 5 * x

print(my_function(3))   # Output: 15
print(my_function(5))   # Output: 25
print(my_function(9))   # Output: 45

🔵 Without return

def square(n):
    res = n * n
    print(res) # prints inside

# Can't use result elsewhere
val = square(4)  # val = None

🟢 With return

def square(n):
    res = n * n
    return res  # sends back

# Can use result elsewhere
val = square(4)  # val = 16
5️⃣ Default Arguments
📌
You can give a parameter a default value using the assignment operator =. If no argument is provided, the default is used.
def calculateTotal(amount, discountPercentage=0):
    discountAmount = discountPercentage / 100 * amount
    return amount - discountAmount

amount = 500

# With explicit discount
totalBillAmount = calculateTotal(amount, 10)
print(totalBillAmount)   # Output: 450.0

# Without discount — uses default (0)
totalBillAmount = calculateTotal(amount)
print(totalBillAmount)   # Output: 500.0
⚠️ Default Argument Rule

Default arguments must come after non-default arguments in the parameter list. You cannot put a default parameter before a required one.

6️⃣ Keyword Arguments
📌
Send arguments using the key=value syntax. The order doesn't matter — values are matched by name, not position.
def my_function(child3, child2, child1):
    print("The youngest child is " + child3)

# Order doesn't matter when using keyword args
my_function(child1="Emil", child2="Tobias", child3="Linus")
# Output: The youngest child is Linus

✅ Advantages

Order doesn't matter · More readable · Reduces argument confusion · Can skip optional args

📝 Note

Keyword arguments are also known as kwargs. The key name must match the parameter name exactly.

7️⃣ Arbitrary Arguments (*args and **kwargs)
*args — Positional
📌
Add a * before the parameter. Function receives a tuple of all positional arguments.
def my_function(*kids):
    print("Youngest: " + kids[2])

my_function("Emil","Tobias","Linus")
# Output: Youngest: Linus

# mysum example
def mysum(*a):
    total = 0
    for ele in a:
        total += ele
    return total

print(mysum(5, 2, 9, 4))  # 20
**kwargs — Keyword
📌
Add ** before the parameter. Function receives a dictionary of all keyword arguments.
def my_function(**kid):
    print("Last name: "+kid["lname"])

my_function(fname="Tobias",
            lname="Refsnes")
# Output: Last name: Refsnes

# mymax example
def mymax(a, *b):
    large = a
    for ele in b:
        if ele > large:
            large = ele
    return large

print(mymax(4,5,7,3,1,8,2))  # 8
📊 Quick Comparison

Positional args

func(1, 2, 3)
Matched by position

*args

func(1,2,3,4,5)
Packed into tuple — any number

**kwargs

func(a=1,b=2)
Packed into dict — named pairs

8️⃣ Passing a List as an Argument
📌
You can send any data type as an argument — strings, numbers, lists, dictionaries, tuples, etc. They are treated as the same data type inside the function.
def my_function(food):
    for x in food:
        print(x)

fruits = ["apple", "banana", "cherry"]
my_function(fruits)
# Output:
# apple
# banana
# cherry

# Can also pass directly
my_function(["mango", "grape"])
9️⃣ The pass Statement
📌
Function definitions cannot be empty. If you need an empty function (placeholder), use the pass statement to avoid a syntax error.
# This causes an error!
def myfunction():
    # IndentationError: expected an indented block

# Correct way — use pass
def myfunction():
    pass   # valid placeholder

# pass is also used in loops, if statements, classes
💡 When to use pass

When you're writing code and want to create a function structure first without filling in the body yet. It lets you write the "skeleton" of your program first and fill in details later.

🔟 Function Returning Multiple Values
📌
Python allows a function to return more than one value by separating them with commas. They are returned as a tuple.
import math

def quadEquation(a, b, c):
    x1 = (-b + math.sqrt(b**2 - 4*a*c)) / (2*a)
    x2 = (-b - math.sqrt(b**2 - 4*a*c)) / (2*a)
    return x1, x2   # returns a tuple

if __name__ == "__main__":
    a = int(input("Enter the value of a: "))
    b = int(input("Enter the value of b: "))
    c = int(input("Enter the value of c: "))
    x1, x2 = quadEquation(a, b, c)  # tuple unpacking
    print(x1, x2)

This program solves the quadratic formula ax² + bx + c = 0 and returns both roots x₁ and x₂. The values are unpacked into two separate variables on the receiving side.

1️⃣1️⃣ Functions as Objects
📌
In Python, functions are first-class objects. Although created differently from normal variables, they can be assigned to variables, passed as arguments, stored in data structures, and referenced by different names.
def multiply(x, y):
    return x * y

a = 4
b = 7

# Assign function to variable (no parentheses!)
operation = multiply

# Call via the variable
print(operation(a, b))   # Output: 28

# Pass function as argument to another function
def apply(func, x, y):
    return func(x, y)

print(apply(multiply, 3, 5))   # Output: 15

🧩 First-Class Functions

Functions can be assigned to variables, passed as arguments, returned from functions, and stored in lists/dicts.

🔄 Function Reference

Writing multiply without () gives you a reference to the function object. Writing multiply() calls it.
